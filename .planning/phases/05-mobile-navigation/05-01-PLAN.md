# Phase 5: Mobile Navigation

## Objective

Add a hamburger menu toggle for mobile viewports (≤968px) so the nav is accessible on all screen sizes. Currently the nav is hidden entirely via `display: none !important` on mobile.

## Execution Context

- **Branch:** feat/mobile-navigation
- **Scope:** `index.html`, all sub-page `*.html`, `assets/css/style.css`, `assets/js/main.js`
- **Out of scope:** Visual redesign of nav, changing nav links, desktop nav behavior

## Context

### Discovery Findings

1. **Nav structure** — All pages share identical nav markup:
   ```html
   <nav class="site-nav">
     <div class="logo"><span>Lewis Whitham</span></div>
     <ul class="nav-links">...</ul>
   </nav>
   ```

2. **Current mobile CSS** — `style.css:2123-2127`:
   ```css
   @media (max-width: 968px) {
     nav { display: none !important; }
   }
   ```
   This completely hides the nav on mobile — no way to navigate.

3. **Nav styling** — `.site-nav` is positioned with `left: 50%; transform: translateX(-50%); top: 1rem;` — pill-shaped floating nav bar with glassmorphism background.

4. **Sub-pages** — Same nav structure, links point to `index.html#section`. Nav toggle must work on all pages.

5. **Breakpoints** — 968px is the mobile breakpoint used throughout the CSS.

### Approach

- Add a hamburger button (`<button class="nav-toggle">`) inside `.site-nav`, hidden on desktop
- On mobile: show the toggle button, hide `.nav-links` by default
- Toggle `.nav-links` visibility when hamburger is clicked (slide-down or fade-in)
- Close menu when a link is clicked
- Use CSS transitions for smooth animation
- Match existing dark theme / glassmorphism aesthetic
